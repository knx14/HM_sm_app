01API 設計と目的 api design

実装に入る前に、各 API が何のために存在するか・何を返すかを 明確にしておきます。ここを理解してから実装することで、Cursor への指示精度が上がり テストケースも自然に書けるようになります。

① 時系列 API — なぜ必要か

農家が「施肥をして土壌が改善されたか」を確認したい場合、 複数の測定日にまたがって CEC 値の推移を見る必要があります。 時系列タブはこの折れ線グラフを表示するために使います。

固定地点は導入しない（Phase 設計の決定事項）ため、 各測定日の「圃場全体の測定点を平均した値」を時系列で返します。 min/max も返すことで「ばらつきの変化」も合わせて確認できるようにします。

GET /api/farms/{farmId}/results/timeseries 時系列タブ用 — 測定日ごとの圃場平均/min/max/count

クエリパラメータ

パラメータ必須デフォルト説明
parameter任意CECresult_values.parameter_name に一致する文字列。Flutter 側から切り替えて使う4種: CEC / CaO / K2O / MgO。未指定は CEC。
推定値は17種類あるが、地図・時系列タブで表示するのは4種のみ。 UIのパラメータ切替タブ（CEC / CaO / K2O / MgO）に対応する4値を対象とします。 各タブを切り替えるたびに Flutter がこのエンドポイントを ?parameter=CEC（または CaO/K2O/MgO）で叩きます。 残り13種は将来的に必要になった時点で対応します。

集計ルール

  • uploads.status = 'completed' のみ対象
  • 集計単位は uploads.measurement_date（同日に複数 upload がある場合も日付で統合）
  • count は upload 数ではなく 測定点数（analysis_results 行数）
  • 該当パラメータのデータがない日は結果から除外

② タイムライン API — なぜ必要か

農家の行動サイクルは「施肥 → 測定 → 効果確認」の繰り返しです。 タイムラインタブはこのサイクルを一本の縦スクロールリストとして見せることで、 「あの施肥の後に CEC が上がったな」という因果関係を直感的に把握できるようにします。

測定結果カードには「効果（delta）」として直前の測定結果との差分を自動計算して表示します。 直前測定との avg 差分を返すことで、農家が変化量を一目で確認できます。

GET /api/farms/{farmId}/timeline タイムラインタブ用 — 測定結果 + 作業ログを日付降順でマージ

並び順

日付降順（新しい順）。同日の場合は測定結果 → 作業ログの順。

values に含めるパラメータ

タイムラインの測定結果カードには CEC / CaO / K2O / MgO の4種 の avg/min/max を含めます。Flutter 側はデフォルトで CEC の avg を表示し、他3種は補足表示に使います。

delta（効果）の算出ルール

  • delta は CEC のみ。CaO/K2O/MgO の delta は返しません（delta の主目的は「土壌改善の全体傾向」の把握のため CEC で十分）
  • 各測定結果アイテムについて、その日の直前の測定日（status='completed' の中で最も近い過去日）の CEC avg との差分を計算
  • 直前測定が存在しない（最初の測定）場合は null

ResultsTimeseriesService を実装

app/Services/ResultsTimeseriesService.php

SQL で測定日ごとに AVG/MIN/MAX/COUNT を集計し、 同じ圃場の work_logs も取得して 1 つの配列にまとめて返します。 

app/Services/ResultsTimeseriesService.php<?php

namespace App\Services;

use Illuminate\Support\Facades\DB;

class ResultsTimeseriesService
{
    /**
     * 指定圃場・パラメータの時系列データを返す。
     *
     * @param  int     $farmId
     * @param  string  $parameter  UI切替4種のいずれか: 'CEC' | 'CaO' | 'K2O' | 'MgO'
     * @return array{
     *   parameter: string,
     *   unit: string|null,
     *   points: array<array{date:string, avg:float, min:float, max:float, count:int}>,
     *   work_logs: array<array{date:string, work_type:string, title:string|null}>
     * }
     */
    public function get(int $farmId, string $parameter = 'CEC'): array
    {
        // ── ① 測定日ごとの avg/min/max/count を集計 ──────────────────
        // uploads.measurement_date 単位でグループ化。
        // status='completed' のみを対象とし、
        // count は upload 数ではなく analysis_results の行数（測定点数）を返す。
        $rows = DB::select('
            SELECT
                u.measurement_date                          AS date,
                ROUND(AVG(rv.parameter_value), 2)           AS avg,
                ROUND(MIN(rv.parameter_value), 2)           AS min_val,
                ROUND(MAX(rv.parameter_value), 2)           AS max_val,
                COUNT(ar.id)                                AS count_points,
                MAX(rv.unit)                                AS unit
            FROM uploads u
            INNER JOIN analysis_results ar ON ar.upload_id = u.id
            INNER JOIN result_values rv
                ON rv.analysis_result_id = ar.id
                AND rv.parameter_name = ?
            WHERE u.farm_id = ?
              AND u.status  = ?
            GROUP BY u.measurement_date
            ORDER BY u.measurement_date ASC
        ', [$parameter, $farmId, 'completed']);

        // ── ② unit を先頭行から取得（全行同じ unit の前提）──────────
        $unit = count($rows) > 0 ? $rows[0]->unit : null;

        // ── ③ 整形 ───────────────────────────────────────────────
        $points = array_map(fn($r) => [
            'date'  => $r->date,
            'avg'   => (float) $r->avg,
            'min'   => (float) $r->min_val,
            'max'   => (float) $r->max_val,
            'count' => (int)   $r->count_points,
        ], $rows);

        // ── ④ 同圃場の作業ログ（グラフの縦線用）─────────────────────
        $workLogs = DB::select('
            SELECT
                work_date   AS date,
                work_type,
                title
            FROM work_logs
            WHERE farm_id = ?
            ORDER BY work_date ASC
        ', [$farmId]);

        $workLogsFormatted = array_map(fn($w) => [
            'date'      => $w->date,
            'work_type' => $w->work_type,
            'title'     => $w->title,
        ], $workLogs);

        return [
            'parameter' => $parameter,
            'unit'      => $unit,
            'points'    => $points,
            'work_logs' => $workLogsFormatted,
        ];
    }
}
なぜ ORM（Eloquent）ではなく生 SQL か。 この集計は複数テーブルを 3 段 JOIN した上で GROUP BY と集計関数を使います。 Eloquent で書けなくはないですが、SQL の方が意図が直接読み取れ、 実行計画のデバッグも容易です。シンプルさを優先して DB::select を使います。

FarmTimelineService を実装

app/Services/FarmTimelineService.php

測定結果と作業ログをそれぞれ取得し、日付でマージして降順に並べます。 測定結果には直前測定との delta を付与します。 

app/Services/FarmTimelineService.php<?php

namespace App\Services;

use Illuminate\Support\Facades\DB;

class FarmTimelineService
{
    /**
     * 指定圃場のタイムライン（測定結果 + 作業ログ）を返す。
     * 日付降順（新しい順）で統合されたアイテム配列。
     *
     * 測定結果の values には CEC / CaO / K2O / MgO の4種を含める。
     * delta は CEC のみ（直前測定の CEC avg との差分）。
     *
     * @param  int    $farmId
     * @return array<int, array>
     */
    public function get(int $farmId): array
    {
        // ── ① 表示対象の4パラメータを定数で管理 ──────────────────────────
        // 推定値は17種類あるが、タイムラインカードに表示するのは以下の4種のみ。
        // 将来追加する場合はこの配列に追記する。
        $targetParams = ['CEC', 'CaO', 'K2O', 'MgO'];

        // ── ② 測定日ごとに4パラメータを一括集計 ─────────────────────────
        // IN句でパラメータ名を絞り込み、CASE WHEN で列に展開する（PIVOT相当）。
        // count_points は CEC の行数で代表させる（4種すべて同じ点数のため）。
        $measurements = DB::select('
            SELECT
                u.measurement_date AS date,
                COUNT(CASE WHEN rv.parameter_name = ? THEN ar.id END)              AS count_points,
                ROUND(AVG(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS avg_cec,
                ROUND(MIN(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS min_cec,
                ROUND(MAX(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS max_cec,
                MAX(CASE WHEN rv.parameter_name = ? THEN rv.unit END)              AS unit_cec,
                ROUND(AVG(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS avg_cao,
                ROUND(MIN(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS min_cao,
                ROUND(MAX(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS max_cao,
                MAX(CASE WHEN rv.parameter_name = ? THEN rv.unit END)              AS unit_cao,
                ROUND(AVG(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS avg_k2o,
                ROUND(MIN(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS min_k2o,
                ROUND(MAX(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS max_k2o,
                MAX(CASE WHEN rv.parameter_name = ? THEN rv.unit END)              AS unit_k2o,
                ROUND(AVG(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS avg_mgo,
                ROUND(MIN(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS min_mgo,
                ROUND(MAX(CASE WHEN rv.parameter_name = ? THEN rv.parameter_value END), 2) AS max_mgo,
                MAX(CASE WHEN rv.parameter_name = ? THEN rv.unit END)              AS unit_mgo
            FROM uploads u
            INNER JOIN analysis_results ar ON ar.upload_id = u.id
            INNER JOIN result_values rv
                ON rv.analysis_result_id = ar.id
                AND rv.parameter_name IN (?, ?, ?, ?)
            WHERE u.farm_id = ?
              AND u.status  = ?
            GROUP BY u.measurement_date
            ORDER BY u.measurement_date ASC
        ', [
            // CASE WHEN のパラメータ（各種 4回ずつ: count, avg, min, max, unit）
            'CEC', 'CEC', 'CEC', 'CEC', 'CEC',
            'CaO', 'CaO', 'CaO', 'CaO',
            'K2O', 'K2O', 'K2O', 'K2O',
            'MgO', 'MgO', 'MgO', 'MgO',
            // IN句
            'CEC', 'CaO', 'K2O', 'MgO',
            // WHERE
            $farmId, 'completed',
        ]);

        // ── ③ delta（直前測定との CEC avg 差分）を計算しながら整形 ──────
        // 昇順で取得しているので、ループ内で prevAvgCec を持ち越すだけで O(n)。
        $measurementItems = [];
        $prevAvgCec = null;

        foreach ($measurements as $row) {
            $avgCec = $row->avg_cec !== null ? (float) $row->avg_cec : null;
            $delta  = ($prevAvgCec !== null && $avgCec !== null)
                ? round($avgCec - $prevAvgCec, 2)
                : null;

            // values: 値が存在するパラメータだけ含める（モデル未実装で null の場合はスキップ）
            $values = [];
            if ($row->avg_cec !== null) {
                $values['CEC'] = ['avg' => (float) $row->avg_cec, 'min' => (float) $row->min_cec, 'max' => (float) $row->max_cec, 'unit' => $row->unit_cec];
            }
            if ($row->avg_cao !== null) {
                $values['CaO'] = ['avg' => (float) $row->avg_cao, 'min' => (float) $row->min_cao, 'max' => (float) $row->max_cao, 'unit' => $row->unit_cao];
            }
            if ($row->avg_k2o !== null) {
                $values['K2O'] = ['avg' => (float) $row->avg_k2o, 'min' => (float) $row->min_k2o, 'max' => (float) $row->max_k2o, 'unit' => $row->unit_k2o];
            }
            if ($row->avg_mgo !== null) {
                $values['MgO'] = ['avg' => (float) $row->avg_mgo, 'min' => (float) $row->min_mgo, 'max' => (float) $row->max_mgo, 'unit' => $row->unit_mgo];
            }

            $measurementItems[] = [
                'type'         => 'measurement',
                'date'         => $row->date,
                'count_points' => (int) $row->count_points,
                'values'       => $values,
                'delta'        => ['CEC' => $delta],  // delta は CEC のみ
            ];

            if ($avgCec !== null) {
                $prevAvgCec = $avgCec;
            }
        }

        // ── ③ 作業ログ取得 ───────────────────────────────────────────
        $workLogRows = DB::select('
            SELECT
                work_date   AS date,
                work_type,
                title,
                detail,
                amount_value,
                amount_unit
            FROM work_logs
            WHERE farm_id = ?
            ORDER BY work_date ASC
        ', [$farmId]);

        $workLogItems = array_map(fn($w) => [
            'type'         => 'work_log',
            'date'         => $w->date,
            'work_type'    => $w->work_type,
            'title'        => $w->title,
            'detail'       => $w->detail,
            'amount_value' => $w->amount_value !== null
                                 ? (float) $w->amount_value
                                 : null,
            'amount_unit'  => $w->amount_unit,
        ], $workLogRows);

        // ── ④ 測定結果 + 作業ログを日付降順にマージ ─────────────────────
        // 同日の場合は measurement を先にする（施肥と測定が同日の場合の表示順制御）
        $all = array_merge($measurementItems, $workLogItems);

        usort($all, function ($a, $b) {
            if ($a['date'] === $b['date']) {
                // 同日: measurement を先 (work_log を後) に並べる
                return $a['type'] === 'measurement' ? -1 : 1;
            }
            // 日付降順
            return $b['date'] <=> $a['date'];
        });

        return ['items' => $all];
    }
}
delta の算出方針（Phase 設計の決定事項を反映）。 delta は CEC のみです。CaO/K2O/MgO の delta は返しません。 昇順ループで $prevAvgCec を持ち越す実装で O(n) で処理できます。
CaO / K2O / MgO のモデルがまだ未実装の場合。 SQL の CASE WHEN で集計した結果が null になるため、 if ($row->avg_cao !== null) の条件分岐でスキップされます。 モデルが追加された時点で自動的に values に含まれるようになります。 Flutter 側は受け取った values キーの存在チェックをして表示を切り替えてください。
