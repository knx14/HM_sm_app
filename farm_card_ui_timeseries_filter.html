FarmController に destroy メソッドを追加

app/Http/Controllers/Api/FarmController.php（編集） 
FarmController.php — destroy メソッドを追加    /**
     * DELETE /api/v1/farms/{farm}
     *
     * 認証ユーザーが所有する圃場を削除する。
     * 圃場に紐づく uploads / analysis_results / result_values は
     * ON DELETE CASCADE か、または手動でカスケード削除する。
     *
     * @param  Farm  $farm  Route Model Binding（自動的に認証ユーザーの圃場か確認する）
     * @return \Illuminate\Http\JsonResponse
     */
    public function destroy(Farm $farm): \Illuminate\Http\JsonResponse
    {
        // 認可: 自分の圃場のみ削除可能
        if ($farm->app_user_id !== $this->authUser()->id) {
            return response()->json(['message' => 'Forbidden'], 403);
        }

        // 関連データのカスケード削除
        // ON DELETE CASCADE が設定されていない場合は手動で削除する
        DB::transaction(function () use ($farm) {
            // uploads → analysis_results → result_values の順に削除
            $farm->uploads->each(function ($upload) {
                $upload->analysisResults->each(function ($ar) {
                    $ar->resultValues()->delete();
                });
                $upload->analysisResults()->delete();
            });
            $farm->uploads()->delete();
            // 作業ログも削除（farm_work_logs または work_logs テーブル）
            if (method_exists($farm, 'workLogs')) {
                $farm->workLogs()->delete();
            }
            $farm->delete();
        });

        return response()->json(['message' => 'deleted'], 200);
    }
リレーションメソッド名を確認してください。 Farm モデルの uploads()、Upload モデルの analysisResults()、 AnalysisResult モデルの resultValues() が定義済みであることを確認してください。 grep -n "function uploads\|function analysis\|function result" app/Models/Farm.php app/Models/Upload.php で確認できます。
DB にカスケード削除設定がある場合。 マイグレーションで ->onDelete('cascade') が設定されていれば $farm->delete() のみで OK です。 grep -rn "onDelete\|cascade" database/migrations/ | grep farm で確認してください。

tests/Feature/FarmDeleteApiTest.php（新規作成） 
tests/Feature/FarmDeleteApiTest.phpclass FarmDeleteApiTest extends TestCase
{
    use RefreshDatabase;

    /** @test */
    public function owner_can_delete_farm(): void
    {
        $user = AppUser::factory()->create();
        $farm = Farm::factory()->create(['app_user_id' => $user->id]);

        $this->authAs($user)
             ->deleteJson("/api/v1/farms/{$farm->id}")
             ->assertStatus(200)
             ->assertJson(['message' => 'deleted']);

        $this->assertDatabaseMissing('farms', ['id' => $farm->id]);
    }

    /** @test */
    public function other_user_cannot_delete_farm(): void
    {
        $owner = AppUser::factory()->create();
        $other = AppUser::factory()->create();
        $farm  = Farm::factory()->create(['app_user_id' => $owner->id]);

        $this->authAs($other)
             ->deleteJson("/api/v1/farms/{$farm->id}")
             ->assertStatus(403);

        $this->assertDatabaseHas('farms', ['id' => $farm->id]);
    }

    /** @test */
    public function unauthenticated_request_is_rejected(): void
    {
        $farm = Farm::factory()->create();
        $this->deleteJson("/api/v1/farms/{$farm->id}")
             ->assertStatus(401);
    }

    /** @test */
    public function related_uploads_are_also_deleted(): void
    {
        $user   = AppUser::factory()->create();
        $farm   = Farm::factory()->create(['app_user_id' => $user->id]);
        $upload = Upload::factory()->create(['farm_id' => $farm->id]);

        $this->authAs($user)
             ->deleteJson("/api/v1/farms/{$farm->id}")
             ->assertStatus(200);

        // 関連 upload も消えていること
        $this->assertDatabaseMissing('uploads', ['id' => $upload->id]);
    }
}

Future<void> _confirmDelete(BuildContext context) async {
  final ok = await showDialog<bool>(
    context: context,
    builder: (_) => AlertDialog(
      title: const Text('圃場を削除しますか？'),
      content: Text(
        '「${widget.farm!.farmName}」とすべての測定データが削除されます。\nこの操作は取り消せません。'),
      actions: [
        TextButton(
          onPressed: () => Navigator.pop(context, false),
          child: const Text('キャンセル'),
        ),
        TextButton(
          onPressed: () => Navigator.pop(context, true),
          style: TextButton.styleFrom(foregroundColor: Colors.red),
          child: const Text('削除する'),
        ),
      ],
    ),
  ) ?? false;

  if (!ok || !context.mounted) return;

  try {
    // FarmRepository.delete() を呼ぶ
    await context.read<FarmRepository>()
        .delete(widget.farm!.id);

    // 圃場一覧に戻る（編集画面と詳細画面を両方ポップする）
    if (context.mounted) {
      Navigator.of(context)
          .popUntil((route) => route.settings.name == '/'
              || route.isFirst);
      // 圃場一覧を再取得
      context.read<FarmListProvider>().fetchFarms();
    }
  } on DioException catch (e) {
    if (context.mounted) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text('削除に失敗しました: ${e.response?.statusCode}')),
      );
    }
  }
}

TimeseriesNotifier に selectedRange と filteredPoints を追加

lib/features/results/presentation/providers/timeseries_notifier.dart（編集） 
timeseries_notifier.dart — selectedRange の追加/// 時系列グラフの表示範囲を表す enum
enum TimeseriesRange {
  all,        // すべて
  oneYear,    // 1年間
  oneMonth,   // 1ヶ月
}

extension TimeseriesRangeLabel on TimeseriesRange {
  String get label => switch (this) {
    TimeseriesRange.all      => 'すべて',
    TimeseriesRange.oneYear  => '1年間',
    TimeseriesRange.oneMonth => '1ヶ月',
  };
}

// TimeseriesNotifier クラスに追加するフィールドとメソッド

  // 追加: 選択中の表示範囲（デフォルトは「すべて」）
  TimeseriesRange selectedRange = TimeseriesRange.all;

  /// 表示範囲でフィルターした points を返す（グラフ描画に使う）
  List<TimeseriesPoint> get filteredPoints {
    final all = data?.points ?? [];
    if (selectedRange == TimeseriesRange.all) return all;

    final now      = DateTime.now();
    final cutoff   = selectedRange == TimeseriesRange.oneYear
        ? now.subtract(const Duration(days: 365))
        : now.subtract(const Duration(days: 30));

    return all.where((p) {
      final d = DateTime.tryParse(p.date);
      return d != null && !d.isBefore(cutoff);
    }).toList();
  }

  /// 表示範囲を変更する（API 再取得は不要）
  void setRange(TimeseriesRange range) {
    if (selectedRange == range) return;
    selectedRange = range;
    notifyListeners();
  }

作業ログ縦線のフィルター対応

work_logs も表示範囲に合わせてフィルターする（_TimeseriesChart 内）// _TimeseriesChart または _TimeseriesView 内で work_logs もフィルター
final filteredWorkLogs = d.workLogs.where((wl) {
  if (n.selectedRange == TimeseriesRange.all) return true;
  final d = DateTime.tryParse(wl.date);
  if (d == null) return false;
  final cutoff = n.selectedRange == TimeseriesRange.oneYear
      ? DateTime.now().subtract(const Duration(days: 365))
      : DateTime.now().subtract(const Duration(days: 30));
  return !d.isBefore(cutoff);
}).toList();
「1ヶ月」で測定点が1点のみになる場合について。 graphic ライブラリは1点のみのデータでも折れ線は引けませんが、 点自体は表示されます。1点のとき最新値だけを大きく表示し、 「グラフを引くには2点以上の測定が必要です」というメッセージを出すとよいでしょう。
