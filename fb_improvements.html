F1パスワードリセットメール — 未登録アカウントにエラー表示 laravel

Laravel バグ

現状、パスワードリセット画面で存在しないメールアドレスを入力してもエラーが出ず、 「メールを送信した」ような表示になっています（Laravel のデフォルト動作）。 未登録アドレスの場合は明示的なエラーメッセージを表示するよう修正します。

セキュリティ上の注意。 一般的に「そのメールは登録されていません」と明示すると、 攻撃者にアカウント存在確認の手段を与えることになります。 本プロダクトは農家向けの業務ツールで一般公開されていないため、 利便性優先でエラー表示する方針とします。

接続ボタンのロジックをトグル（接続 / 切断）に変更

lib/features/measure/presentation/measurement_session_screen.dart（編集）

接続チップ（または接続ボタン）の onTap を以下のように変更します。 既存の connectSensor() の呼び出しをラップして、 接続済みの場合は切断処理を呼ぶようにします。 

接続チップの onTap をトグルに変更// 変更前
onTap: n.isSensorConnected ? null : n.connectSensor,

// 変更後: 接続済みなら切断、未接続なら接続
onTap: n.isConnecting ? null : // 接続処理中は無効
    n.isSensorConnected
        ? () => n.disconnectSensor()   // 接続済み → 切断
        : () => n.connectSensor(),     // 未接続 → 接続

// チップのラベルも状態に応じて変える
label:    n.isSensorConnected ? '✓ 接続中' : '接続',
subLabel: n.isSensorConnected ? 'タップで切断' : 'センサ',

disconnectSensor() メソッドを MeasurementSessionScreen（または相当する Notifier）に追加します。 

disconnectSensor() メソッドを追加Future<void> disconnectSensor() async {
  try {
    // 既存の USB シリアル接続を閉じる処理（実際の変数名に合わせること）
    await _usbPort?.close();
    _usbPort = null;
  } catch (_) {}
  setState(() {
    isSensorConnected = false;
    isBgDone          = false;  // 切断したら BG もリセット
  });
}
切断時の BG リセットについて。 センサーを切断したら BG 完了フラグもリセットします。 再接続時に再度 BG 測定が必要になりますが、これは正しい仕様です（別のセンサーを繋ぐ場合もある）。

F6測定地点補正を 1m 単位で行えるようにする flutter

Flutter UX
原因の推定。 現行実装では地図タップで _confirmedLocation を直接更新していますが、 Google Maps の onTap コールバックは画面上のピクセル座標を LatLng に変換するため、 ズームレベルが低いと同じピクセルが数十メートル以上の範囲をカバーしてしまいます。 解決策は「最大ズームに強制する」または「方向キー（矢印ボタン）で 1m ずつ移動する」の 2 通りです。 両方組み合わせることを推奨します。

補正モード時に最大ズームを強制する

lib/features/measure/presentation/measurement_session_screen.dart（編集） 
補正モード開始時（startPinCorrection 呼び出し後）に最大ズームへ移動void startPinCorrection(int spotId) {
  setState(() => _correctingSpotId = spotId);

  // 補正対象ピンの位置に最大ズームで移動
  final pin = _spots.firstWhereOrNull((s) => s.id == spotId);
  if (pin != null && _mapController != null) {
    _mapController!.animateCamera(
      CameraUpdate.newCameraPosition(CameraPosition(
        target: LatLng(pin.position.latitude, pin.position.longitude),
        zoom:   21,  // Google Maps の最大ズーム（衛星: 20〜21）
      )),
    );
  }
}

// 1m ずつ LatLng を動かすヘルパー
// 緯度 1° ≒ 111,000m。経度は cos(lat) 倍
void _nudge(int latDir, int lngDir) {
  final spot = _spots.firstWhereOrNull((s) => s.id == _correctingSpotId);
  if (spot == null) return;

  const mPerDeg = 111000.0;
  final latDelta = latDir / mPerDeg;
  final lngDelta = lngDir / (mPerDeg * cos(spot.position.latitude * pi / 180));

  final newPos = LatLng(
    spot.position.latitude  + latDelta,
    spot.position.longitude + lngDelta,
  );
  setState(() => spot.position = newPos);
  // _confirmedLocation も更新（既存の位置補正ロジックと合わせること）
  _confirmedLocation = newPos;
}

// 矢印ボタンウィジェット
class _ArrowButton extends StatelessWidget {
  const _ArrowButton(this.icon, this.onTap);
  final IconData   icon;
  final VoidCallback onTap;
  @override
  Widget build(BuildContext _) => GestureDetector(
    onTap: onTap,
    child: Container(
      width: 44, height: 44,
      decoration: BoxDecoration(
        color: Colors.white.withValues(alpha: 0.9),
        shape: BoxShape.circle,
        boxShadow: [BoxShadow(
          color: Colors.black.withValues(alpha: 0.2),
          blurRadius: 4)],
      ),
      child: Icon(icon, size: 20),
    ),
  );
}

Marker を番号付きカスタムアイコンに変更

lib/features/measure/presentation/measurement_session_screen.dart（編集）

現状は BitmapDescriptor.defaultMarkerWithHue の標準ピンを使っているため大きく見えます。 Canvas でサイズを抑えた番号付き円アイコンを生成して置き換えます。 

番号付き小型カスタムマーカーアイコンを生成するヘルパーimport 'dart:ui' as ui;
import 'dart:typed_data';

/// 番号付き小型円アイコンを生成する。
/// [number]: 表示する番号（1始まり）
/// [status]: 測定中=赤 / 完了=黄 / 推定完了=緑
/// [size]:   円の直径 px（推奨: 48〜56）
Future<BitmapDescriptor> createNumberedMarker(
    int number, PinStatus status, {double size = 52}) async {
  final color = switch (status) {
    PinStatus.measuring => const Color(0xFFC0392B),
    PinStatus.measured  => const Color(0xFFE67E22),
    PinStatus.completed => const Color(0xFF27AE60),
  };

  final recorder = ui.PictureRecorder();
  final canvas    = Canvas(recorder);
  final r         = size / 2;

  // 円背景
  canvas.drawCircle(
    Offset(r, r), r,
    Paint()..color = color,
  );
  // 白い縁取り
  canvas.drawCircle(
    Offset(r, r), r,
    Paint()..color = Colors.white..style = PaintingStyle.stroke..strokeWidth = 2,
  );
  // 番号テキスト
  final tp = TextPainter(
    text: TextSpan(
      text: '$number',
      style: TextStyle(
        color: Colors.white,
        fontSize: size * 0.42,
        fontWeight: FontWeight.bold,
      ),
    ),
    textDirection: TextDirection.ltr,
  )..layout();
  tp.paint(canvas, Offset(r - tp.width / 2, r - tp.height / 2));

  final img   = await recorder.endRecording().toImage(size.toInt(), size.toInt());
  final bytes = await img.toByteData(format: ui.ImageByteFormat.png);
  return BitmapDescriptor.fromBytes(bytes!.buffer.asUint8List());
}

// 使用例（Marker 生成時）
final icon = await createNumberedMarker(pin.index, pin.status);
Marker(
  markerId: MarkerId('pin_${pin.index}'),
  position: LatLng(pin.latitude, pin.longitude),
  icon:     icon,
  // InfoWindow は F8 で無効化するため省略
)
アイコン生成は非同期処理です。 createNumberedMarker は Future を返すため、 ピンを追加するたびに await してからマーカーセットに入れてください。 パフォーマンスが気になる場合はキャッシュ（Map<int, BitmapDescriptor>）を使ってください。

measurement_state_provider.dart — 画面をまたいで保持すべき状態のみ管理import 'package:flutter/foundation.dart';
import 'package:google_maps_flutter/google_maps_flutter.dart';

/// アプリ全体で保持する測定状態（画面離脱後も維持）。
/// - ピン一覧（当日の測定地点）
/// - USB センサーの接続状態（F10 と共通）
/// - BG 完了フラグ
class MeasurementStateProvider extends ChangeNotifier {

  // ピン一覧
  final List<MeasurementPin> pins = [];

  // センサー接続状態（F10 で USB ポートもここに持つ）
  bool    isSensorConnected = false;
  bool    isBgDone          = false;

  // 同日の測定か判断するための基準日
  String? _sessionDate;  // "YYYY-MM-DD"

  /// 新しい測定日を開始するとき（日付が変わったらピンをリセット）
  void startSession(String today) {
    if (_sessionDate != today) {
      pins.clear();
      _sessionDate = today;
    }
    notifyListeners();
  }

  /// ピンを追加
  void addPin(MeasurementPin pin) {
    pins.add(pin);
    notifyListeners();
  }

  /// ピン状態を更新（測定中→完了→推定完了）
  void updatePin(int index, PinStatus status, {double? cec}) {
    final i = pins.indexWhere((p) => p.index == index);
    if (i < 0) return;
    pins[i] = pins[i].copyWith(status: status, cec: cec);
    notifyListeners();
  }

  /// ピンを削除
  void removePin(int index) {
    pins.removeWhere((p) => p.index == index);
    notifyListeners();
  }

  /// 接続状態を更新（F10 と共用）
  void setSensorConnected(bool v) {
    isSensorConnected = v;
    if (!v) isBgDone = false;  // 切断したら BG もリセット
    notifyListeners();
  }

  void setBgDone(bool v) {
    isBgDone = v;
    notifyListeners();
  }
}

MeasurementSessionScreen を Provider から読み書きするように変更

既存の MeasurementSessionScreen 内で setState(() => _pins.add(...)) のようにローカル State を更新している箇所を、 context.read<MeasurementStateProvider>().addPin(...) に置き換えます。 ウィジェットで表示する際は context.watch<MeasurementStateProvider>().pins を使います。

既存のコード量が多い場合の進め方。 MeasurementSessionScreen のすべての State を一度に移行しようとせず、 「ピン一覧」と「接続状態」の 2 つだけを先に移行して動作確認してください。 BG のプログレスなど画面内だけで使う状態はローカル State のまま残して問題ありません。
